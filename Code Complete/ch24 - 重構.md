# 重構
* 軟體演化的基本原則: 提升程式的內在品質
* 改善系統的方法之一是提升模組性
	* 對子程式定義完善並且良好命名，如果能將某段程式獨立出來變成另一個子程式來讓程式更清晰，那就把它獨立出來
## 重構的理由
* 重複的程式碼
* 冗長的子程式
	* 超過一個螢幕長度的子程式是必須被重構的
* 迴圈過長或是巢狀結構過深
* 內聚性太差的類別
* 類別介面沒有提供一致的抽象介面
* 擁有太多參數的函數
* 當對類別內的修改往往只改變了某個部分
	* 代表類別內部相關的部分可以在個別拆開成其他子程式
* 變化導致對多個類別的相同修改
